2020 NFL Ratings... and Why Choice of Loss Function Matters

January 14, 2021

In this project, I create 2020 NFL rankings based on the method described in Mathletics by Wayne Winston. I do so by scraping NFL game data in Python and creating team ratings using Microsoft Excel’s Solver tool. The relevant Python code and Excel files are all included in the GitHub Repo so that the reader may play around with the ratings if they like. The code is also thoroughly commented so that he/she may change things around as appropriate, and the Excel formulas are all pretty straightforward. The goal of this post is to:

  1. Briefly outline how the rankings work
  2. Explain how they can be used to predict game results
  3. Note the importance of choice of loss function

1.

Below is the set of rankings I created by minimizing a sum of squared errors (more on what this means below). Teams are sorted by highest rating to lowest, and the interpretation is that a team’s rating is the number of points by which they’re better (or worse) than the completely average NFL team. So, for example, the Saints rating of 9.69 means they are 9.69 points better than the average NFL team. Why does this work? Because the average of all 32 ratings is 0.

While Winston explains the ranking method in a much more detailed manner, the idea is pretty simple. We collect NFL game-by-game data and calculate the margin of victory for the home team. The example below comes from the csv nfl_game_data.csv in the data/ folder.

Once we have these margins, we create pilot ratings for each team. Think of these as guesses, or initial ratings. They could be anything (I generated them randomly in Python), but they must average to 0. We also input a trial home field advantage (3 is a good starting point). Next, given these ratings, we use them along with our home field advantage guess to create a predicted margin for each game. This prediction is predicted margin = home field advantage + home team rating - away team rating. In Excel, you will have to use VLOOKUP to make this work, and the reader can see the file nfl_rankings.xlsx for how I do it.

The rest is pretty easy. Calculate the prediction error as error = actual margin - predicted margin for each game. Next, square the errors and take their sum. This sum is called the sum of squared errors (SSE). Now, all that needs to be done is use the Excel Solver to find the set of ratings that minimizes the SSE. This makes sense, right? If we can tweak the team ratings to make our predicted margins close to the actual margins, then our errors are small. This is what we want, and this is how I created these rankings. The image below is a screenshot of my Solver window, where cell K35 contains the “objective”, the SSE, C2:C33 include the team ratings, and F7 contains the home edge. The constraint in C34 is that the average of the ratings equals 0. This is easier to follow with the nfl_rankings.xlsx pulled up.

2.

So, now that we have team ratings, how are they useful? We can actually use them to forecast future games, and let’s do so for this weekend’s playoff games. It’s really easy: our predictions are home team margin of victory = home field advantage + home team rating - away team rating. A negative number means we predict the home team to lose, while a positive one means we predict the home team to win. Let’s see this in action using the above ratings. (Note that home field advantage is worth essentially 0 points).

2020 Divisional Round Predictions

Game 1; LAR @ GB: GB margin of victory = 0 + 7.64 - 5.91 = 1.73. Game 2; BAL @ BUF: BUF margin of victory = 0 + 7.55 - 8.31 = -0.36. Game 3; CLE @ KC: KC margin of victory = 0 + 6.79 - (-1.63) = 8.42. Game 4; TB @ NO: NO margin of victory = 0 + 9.70 - 9.28 = 0.42.

We should get some pretty good games this weekend!

3.

In this section, I’ll say a bit about my biggest takeaway from this mini-project: choice of loss function matters. But first, what’s a loss function? In one sentence, it’s our sense of error–our sense of how far our prediction varies from the actual result. And we can choose it! The importance of one’s choice of loss function is one of the first things we learn in a modeling class, but it can feel a bit abstract. This project has been illuminating for me because it has revealed some concrete examples of this, so here goes.

In the above rankings, it might be surprising that CLE, a playoff team(!), comes in at 20th overall. How could this be?! They’re supposed to be good. The key part: if one examines CLE’s schedule this year, he/she might notice a couple blowout losses: in Week 1, they lost 38-6 to BAL, and in Week 6, they lost 38-7 to PIT… revenge is sweet. These blowout losses mean that the actual margin and predicted margin vary greatly, thereby contributing to a large error. This is where the loss function comes in. As I noted earlier, we SQUARE each error because our loss function is the squared loss. So this error gets blown up to something really big. For this reason, CLE gets penalized very hard for these bad losses, and the optimal solution will therefore rank them low.

Okay, that makes sense, but what can we do about it? A common alternative is to use the absolute loss, meaning we take the absolute value of the error instead of its square. So, instead of minimizing the sum of squared errors, we minimize the sum of absolute errors. Doing so yields the following rankings.

Suddenly, CLE has jumped from 20th to 16th, which makes more sense. (I know there are some other teams who’ve moved a lot, but let’s focus on CLE). Again, the upshot here is that, under the absolute loss, blowout losses will not lead to massive penalties like they will under the squared loss.

Last, another approach is to stick with the squared loss but remove these “outlier” games. The rankings below are generated from the set of games with CLE’s Week 1 and Week 6 blowouts removed along with their Week 16 game that they played without their entire WR core. I also removed PIT’s and KC’s Week 17 games, as PIT rested Big Ben and KC did not play their starters.

Wow, the Browns are all the way up to 11 now. In addition, note that although I removed PIT’s Week 17 contest, they actually dropped a spot and their rating decreased since their blowout victory versus CLE in Week 6 was removed… so they are rewarded much less. As for KC, they acheive their highest ranking of all three sets of rankings at 3rd overall.

Visually, to see the importance of loss function, here is a plot that shows the biggest “jumps” from teams based on whether the squared or absolute loss was used. To make the plot, a jump greater than 5 was required. Remember that a lower rank is better.

That’s it! I will leave my offense and defense rankings here as well, but the reader will have to read Mathletics to see how they work. They really work the same way, but instead of the error being margin of victory, the error includes points scored and points allowed terms. Finally, the Jupyter Notebooks provided in the GitHub Repo include more instructions and discussion (especially nfl_loss_func_analysis.ipynb), but this wall of text is enough for here.
